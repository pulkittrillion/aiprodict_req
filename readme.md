# AI Product Requirement Generator (n8n + DeepSeek + Cloudflare Tunnel)
![Project diagram](https://i.ibb.co/v4tVJN7V/image.png)

This project is an **AI Product Requirement Document (PRD) generator** built with an **n8n workflow** hosted on a **VPS**, exposed to the internet via a **Cloudflare (trycloudflare) tunnel**, and triggered by a simple **HTML frontend** (web form).

When a user submits a product name + description, the HTML page sends a **POST** request to the n8n webhook, the workflow calls **DeepSeek** to generate a PRD, and the result is returned back to the browser.

---

## What we built

### High-level flow
1. **Frontend (index.html)** collects:
   - `productName`
   - `productDescription`
2. Frontend sends a **POST** request to the n8n webhook:
   - `https://<your-tunnel-url>/webhook-test/prd-generator`
3. **n8n Webhook** receives the payload and triggers workflow execution.
4. **n8n HTTP Request node** calls DeepSeek:
   - `POST https://api.deepseek.com/chat/completions`
5. **n8n Respond to Webhook** returns the DeepSeek response text to the browser.
6. Frontend displays the generated PRD.

---

## Architecture / Components

- **VPS (Ubuntu/Linux assumed)**:
  - Hosts **n8n**
  - Runs workflow and receives webhook requests
- **Cloudflare Tunnel (trycloudflare quick tunnel)**:
  - Exposes your local n8n port (default `5678`) to a public HTTPS URL
- **n8n Workflow**:
  - Webhook → DeepSeek API call → Respond to Webhook
- **Frontend (HTML)**:
  - A standalone `index.html` file with a form + fetch call

## Prerequisites

- VPS with public access
- n8n running on the VPS (default port: `5678`)
- Cloudflare tunnel tool available on the VPS (`cloudflared`)
- A **DeepSeek API key**

---

## VPS Setup (n8n host)

### 1) Ensure n8n is running
- n8n should be reachable locally on:
  - `http://localhost:5678`

### 2) Confirm n8n UI (optional)
- From the VPS itself you should be able to access:
  - `http://localhost:5678`

If needed, start n8n in whatever way your workshop setup used (systemd/docker/manual).

---

## Cloudflare Tunnel Setup (trycloudflare)

> The tunnel URL generated by `trycloudflare.com` is **temporary**. Restarting typically gives a **new random URL**, so you must update your frontend.

### 1) Start the tunnel (from VPS)
Assuming n8n is running on port `5678`:

```bash
cloudflared tunnel --url http://localhost:5678
